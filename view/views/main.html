Technology Stacks

AngularJS and Bootstrap built Web Components, Views and Applications with configuration and data from backend. Embedded Undertow HTTP 2.0 server with non-blocking, asynchronous architecture. Hazelcast Distributed cache and Document-Graph NoSQL database OrientDb. Light Rule Engine glues everything together and command/event sourcing along with CQRS makes framework scalable and supports events replay anywhere.

Components

Component is the smallest deployment unit in the framework and it contains Web Component built with AngularJS, configurations data in database, a set of rules built in Java. Components are separate with data so that it can be open sourced and reused anywhere and it can only be alive as part of the view which can have several components and other contents. Angular schema form is one of the components.

Learn More »

Views

Views are also called pages and it consists several components and static contents. It is working independently with other views except sharing data and routing to other views. Like component, it can be deployed independently and the dependencies and versions are managed by graph database through configuration. Views can also have rules to handle business logic and manage components within.

Learn More »

Applications

An application contains several views linked by routing rules and it provides user experience on certain function on a site. It can be deployed separately and manages views dependencies and versions. One web site can have several applications to hundreds of applications. Blog, Forum and News are common applications already implemented in the framework; however, users can choose to other ui experience.
